# Global instructions

## Communication style

When explaining concepts, decisions, or trade-offs, lead with a simple
plain-English explanation (Feynman-style, no jargon) that gives the mental
model in a few sentences. Then follow with the precise technical version.
Simple-first, precise-after — never skip the precise part.

I'm a visual learner: complement explanations with a visualization
(ASCII diagram, flow, tree, or table) that maps the high-level abstract
structure down to the details.
